【计算机网络】第一章：概述

发表于 2020-03-14 更新于 2020-03-24 分类于 计算机网络

计算机网络的性能

1、速率

连接在计算机网络上的主机在数字信道上传送数据位数的速率，也称data rate & bit rate，单位是b/s, kb/s, Mb/s, Gb/s。和正常理解的网速关系是 网速÷8 = 速率

2、带宽

数据通信领域中，数字信道所能传输的最高速率，单位是b/s，kb/s，Mb/s，Gb/s

3、吞吐量

即单位时间内通过某个网络的数据量，单位是b/s，Mb/s等

4、时延

分为发送时延、传播时延、处理时延和排队时延。

发送时延 = 数据块长度（比特）/ 信道带宽（比特/秒）

传播时延 = 信道长度（米）/ 信号在信道上的传播速率（米/秒）

处理时延，即网络结点存储转发处理时间；排队时延，网络结点缓存队列排队时间

5、时延带宽积

时延带宽积 = 传播时延 x 带宽，用于确认有多少数据在链路上

6、往返时间RRT（Round-Trip Time）

从发送方发送数据开始，到发送方收到接收方确认

7、利用率

信道利用率：有数据通过时间 / (有+无)数据通过时间

网络利用率：信道利用率加权平均值

网络当前时延D = 网络空闲时时延D0 / (1 - 信道利用率U)

计算机网络的体系结构

基本概念

ISO：国际标准化组织

OSI/RM：互联网法律上的国际标准

TCP/IP Suite：因特网事实上的国际标准

NetWork Protocols：数据交换遵守的规则、标准或约定

网络体系结构：计算机网络各层及其协议的集合

OSI参考模型（从高到低

  • 应用层——所有能产生网络流量和与用户交互的程序
  • 表示层——在传输之前是否进行加密 或 压缩 处理，二进制或ASCII码表示（开发人员需要考虑的层
  • 会话层——服务端和客户端建立的会话（查木马netstat -nb，看需求端和网站之间的连接
  • 传输层——可靠传输（建立会话），流量控制，不可靠传输（一个数据包即可，不需要建立会话，例如向DNS查询网站IP地址）
  • 网络层——负责选择最佳路径，规划IP地址(ipv4和ipv6变化只会影响网络层)
  • 数据链路层——帧的开始和结束，还有透明传输，差错校验(纠错由传输层解决)
  • 物理层——定义网络设备接口标准，电气标准(电压)，如何在物理链路上传输的更快

OSI 网络排错与网络安全

每一层都为上一层提供服务，一旦某一层崩了，上面所有层都崩了，所以排查问题从最上层(物理层)排查；

1、物理层故障

查看链接状态，发送和接收的数据包数值（网线接口之类的

2、数据链路层故障

Mac地址冲突，2个一样的Mac地址同时访问；ADSL(非对称数字用户线路)欠费；两端的接口网速没有协商一致；计算机连接到其它VLAN(虚拟互联网)中。

3、网络层故障

配置错误的IP地址，子网掩码，网关；数据有没有通过各个网关到达指定位置；防火墙设置

4、应用层故障

SQL注入漏洞 上传漏洞 分布式攻击（密码学DNS有讲过
